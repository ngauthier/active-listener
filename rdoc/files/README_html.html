<h1>Active Listener: Event listening and firing system for ruby</h1>

<h2>Description</h2>

<p>This gem allows you to define tasks that can be run at specific time intervals. In the future, we plan to allow event listening and firing.</p>

<h2>Installation</h2>

<h3>Latest Stable</h3>

<pre>sudo gem install ngauthier-active-listener </pre>

<h3>Cutting Edge</h3>

<pre>git clone git@github.com:ngauthier/active-listener.git active-listener cd active-listener rake gem install </pre>

<h2>Configuration</h2>

<h3>Add a config file</h3>

<p>In your rails project, add a config file like this:</p>

<pre><code>— 

  tasks:
    - task: my:rake:task
      period: 5

</code></pre>

<p>This instructs active-listener to run "rake my:rake:task" every 5 seconds.</p>

<p>Usually, these config files go in RAILS_ROOT/config/active-listener.yml.</p>

<h3>Include an initializer</h3>

<p>Create a file in the initializers directory, like RAILS_ROOT/config/initializers/active-listener.rb that has this in it:</p>

<pre><code>require ‘active-listener’ ActiveListener.autostart( 

  :config =&gt; File.join(RAILS_ROOT, 'config', 'active-listener.yml'),
  :log_file =&gt; File.join(RAILS_ROOT, 'log', 'active-listener.log'),
  :pid_file =&gt; File.join(RAILS_ROOT, 'log', 'active-listener.pid'),
  :rake_root =&gt; File.join(RAILS_ROOT)

) </code></pre>

<p>This will use the config file "config/active-listener.yml". It will put the log file in "log/active-listener.log" and the pid file for tracking the process in "log/active-listener.pid".</p>

<h2>Usage</h2>

<p>Active Listener will automatically start whenever the rails environment is loaded. It will replace an existing instance of active-listener if the pid file is the same and there is one running.</p>

<p>So, you don‘t need to run the executable, it will "autostart" thanks to the initializer. Keep in mind this means that it will be running during your tests, development server, production, etc. I am working on "environmentalizing" the gem.</p>

<p>If you want to stop active-listener, run:</p>

<pre>active-listener —stop log/active-listener.pid </pre>

<p>Make sure to point it to your pid file.</p>
